# Implementation Plan: Contract Suggestion Examples & Type Fix

**Branch**: `005-contract-suggestion-data` | **Date**: 2026-03-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-contract-suggestion-data/spec.md`

## Summary

Fix contract type misclassification bug (car rental → money lending) by replacing naive substring matching with word-overlap scoring + LLM verification fallback. Add field suggestion examples during contract creation by storing LLM-generated sample data per template in Supabase. Provide a `seed-suggestions` CLI command to populate sample data.

## Implementation Phases

### Phase 1: Fix Contract Type Resolution (P1 — US1)

**Files**: `interactive_chat.py`

1. Replace substring matching in `_resolve_contract_type()` with word-overlap scoring:
   - Tokenize both input and each template display_name into word sets
   - Score = |intersection| / |template_words| (Jaccard-like)
   - Return best match if score > 0.6 and clearly best (gap > 0.1 from #2)
2. If low confidence (score ≤ 0.6 or top-2 close), delegate to `_detect_contract_type_with_llm()`
3. If LLM returns None, return None → triggers "ask user to choose" flow
4. Add logging for resolution decisions (which method, score, chosen type)

### Phase 2: Add Sample Data Infrastructure (P2 — US3)

**Files**: `005_sample_data.sql`, `supabase.py`, `suggestion_seeder.py`, `cli/main.py`

1. Migration: `ALTER TABLE contract_templates ADD COLUMN sample_data JSONB`
2. DB methods: `update_template_sample_data(type, data)`, `get_templates_needing_seed()`
3. `SuggestionSeeder` service:
   - Load template + field definitions
   - Build LLM prompt asking for 2-3 realistic Vietnamese examples per field
   - Parse JSON response → validate structure → save to DB
4. CLI command `seed-suggestions` with `--type`, `--force`, `--status` flags

### Phase 3: Show Suggestions in Field Questions (P1 — US2)

**Files**: `interactive_chat.py`, `api/routes/contract.py`, `api/schemas.py`

1. In `_process_field_answer()`: after loading field, check `template.sample_data[field.name]`
   - If exists: append example to question text (`💡 Ví dụ: {example}`)
   - If not: show question as-is (graceful fallback)
2. In API `ContractFieldItem`: add optional `suggestions` field
3. In `_field_to_item()`: populate suggestions from template sample_data
